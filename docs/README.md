# Angular Self Documentation

This folder is your personal Angular knowledge base.

Goal:
- Write each topic in your own words.
- Keep it very simple.
- Revisit and refine regularly.
- Use Feynman Technique: explain as if teaching a beginner.

## Important Files
- **[DOCUMENTATION-TONE-GUIDE.md](DOCUMENTATION-TONE-GUIDE.md)** → Read this first! It defines how all documentation should be written in your style.

## Structure
- `fundamentals/` → short definitions and core concepts
- `roadmap/` → what to study from beginner to senior

## How to use
1. Read one file.
2. Rewrite the definition in your own words.
3. Add one real example from your project.
4. Add one mistake to avoid.
5. Review weekly.

## When Adding New Topics
Always refer to [DOCUMENTATION-TONE-GUIDE.md](DOCUMENTATION-TONE-GUIDE.md) before writing to maintain consistent style.
