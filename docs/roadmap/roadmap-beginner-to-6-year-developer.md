# Angular Roadmap: Beginner to 6-Year Developer

Use this as a checklist. Write each topic in your own words.

## Stage 1 — Beginner (0 to 6 months)
- HTML, CSS, JavaScript basics
- TypeScript fundamentals (types, interfaces, classes, generics)
- What Angular is, project structure, CLI
- Components, templates, data binding
- Directives (`*ngIf`, `*ngFor`)
- Pipes basics
- Input/Output communication
- Services and basic dependency injection
- Basic routing and route params
- Basic forms (template-driven + intro reactive forms)

## Stage 2 — Junior (6 to 18 months)
- Reactive forms deeply
- Custom validators
- HTTP client, APIs, interceptors
- Observables and RxJS essentials (`map`, `switchMap`, `mergeMap`, `catchError`)
- Component lifecycle hooks
- Parent-child and shared-state patterns
- Route guards, resolvers, lazy loading
- Reusable components and shared modules
- Error handling patterns
- Unit testing basics (components/services)

## Stage 3 — Mid-Level (1.5 to 3 years)
- Advanced RxJS patterns
- State management (NgRx, Signals-based patterns)
- Change detection strategy and performance tuning
- Advanced routing architecture
- Feature-based architecture and domain separation
- Security basics (XSS, CSRF, auth flows, token handling)
- Accessibility (a11y) fundamentals
- Testing strategy (unit + integration)
- CI basics for Angular projects
- SSR/SSG basics (Angular SSR)

## Stage 4 — Senior Track (3 to 6 years)
- Scalable architecture design for large Angular apps
- Monorepos and workspace strategy (Nx or similar)
- Advanced performance profiling and optimization
- Microfrontend concepts
- Advanced security practices and threat modeling
- Design systems and reusable UI libraries
- Strong testing culture (component harnesses, e2e strategy)
- Release strategy, observability, monitoring
- Technical leadership and code review standards
- Mentoring, documentation quality, team enablement

## Cross-cutting topics (always improve)
- Clean code and refactoring
- SOLID principles and design patterns
- Git workflows and collaboration
- Debugging techniques
- API design understanding (REST/GraphQL concepts)
- Browser internals and rendering basics
- Build tooling and bundling basics
- Communication and system design thinking

## Feynman template for every topic
For each topic, write:
1. What is it?
2. Why do we need it?
3. Real project example
4. Common mistakes
5. How to explain to a beginner in 3 lines
